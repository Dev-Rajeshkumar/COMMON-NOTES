# Conventional Commit Message Guidelines

| **Prefix**  | **Description**                                         | **Example Commit Message**                                             |
|-------------|---------------------------------------------------------|------------------------------------------------------------------------|
| **Feat:**   | A new feature or addition to the codebase               | `Feat: Add new router for authentication endpoints`                    |
| **Fix:**    | A bug fix                                                | `Fix: Correct validation error in login form`                          |
| **Docs:**   | Changes to documentation                                 | `Docs: Update README with API usage examples`                          |
| **Style:**  | Code style changes (formatting, missing semicolons, etc.)| `Style: Fix code indentation in UserService.ts`                        |
| **Refactor:** | Code refactoring without adding features or fixing bugs | `Refactor: Simplify query logic in organization service`               |
| **Chore:**  | Maintenance tasks or auxiliary changes (config updates, etc.) | `Chore: Update dependencies and npm scripts`                        |
| **Perf:**   | Code changes that improve performance                    | `Perf: Optimize database query performance for user retrieval`          |
| **Test:**   | Adding or modifying tests                                | `Test: Add unit tests for role-based access control`                    |
| **Revert:** | Reverting a previous commit                              | `Revert: Revert "Feat: Add new subscription service"`                   |
| **Build:**  | Changes that affect the build system or external dependencies | `Build: Update webpack config for production build optimization`    |
| **Ci:**     | Continuous integration-related changes                   | `Ci: Fix CircleCI build pipeline configuration`                        |
